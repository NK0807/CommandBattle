# CommandBattle (コマンドバトル)

Javaで作成した、コンソール（ターミナル）上で動作する1対1のコマンドバトルゲームです。
オブジェクト指向の学習用プロジェクトとして開発しました。

## 🎮 ゲーム概要
勇者（プレイヤー）と魔王（CPU）がターン制で戦います。
先に相手のHPを0にした方が勝利です。

## ✨ 特徴・実装した機能
* **オブジェクト指向設計**
    * `Character` 抽象クラスによる共通仕様の定義
    * `Hero`, `Enemy` クラスへの継承
    * ポリモーフィズムを利用した攻撃ロジックの切り替え
* **ランダム要素**
    * `java.util.Random` を使用したダメージのブレ幅（乱数）
    * 10%の確率で発生する「会心の一撃」
* **ゲームロジック**
    * 独自の攻撃演出（勇者は会心あり、魔王は固定ダメージの業火など）

## 🛠 開発環境
* 言語: Java
* IDE: Eclipse / VS Code (Codespaces)

## 📚 今回学んだこと (Key Learnings)

### 1. オブジェクト指向プログラミング (OOP)
* **カプセル化 (Encapsulation):**
    * フィールドを `private/protected` にし、メソッドを通じてアクセスすることでデータを保護する設計を実践。
* **継承 (Inheritance):**
    * `Hero` と `Enemy` の共通部分を親クラス `Character` にまとめることで、コードの重複を排除し、保守性を向上。
* **多態性 (Polymorphism):**
    * 親クラス型 (`Character`) で引数を受け取り、実体 (`Hero`/`Enemy`) に応じて異なる振る舞い（会心の一撃や業火など）をさせる柔軟な実装。

### 2. 抽象クラスと抽象メソッド (Abstract)
* **抽象クラス (`abstract class`):**
    * 「キャラクター」という概念は直接インスタンス化できないように制約をかけ、設計上の安全性（誤用防止）を確保。
* **抽象メソッド (`abstract method`):**
    * 「攻撃する (`attack`)」という機能は必須としつつ、その具体的な処理内容は各サブクラスに強制的に実装させる設計を学習。

### 3. リファクタリングとGit運用
* **リファクタリング:**
    * 一度動くコードを作った後、継承や抽象化を用いて「動きは変えずにコードをきれいにする」プロセスを経験。
* **Gitフロー (GitHub Flow):**
    * `main` ブランチを直接触らず、機能ごとに `feature/xxx` や `refactor/xxx` ブランチを切って開発し、マージする手法を実践。
